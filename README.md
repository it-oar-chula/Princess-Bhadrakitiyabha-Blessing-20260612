# เว็บร่วมลงนามถวายความอาลัย

สมุดลงนามถวายความอาลัยออนไลน์ แด่ สมเด็จพระเจ้าลูกเธอ เจ้าฟ้าพัชรกิติยาภา นเรนทิราเทพยวดี
กรมหลวงราชสาริณีสิริพัชร มหาวัชรราชธิดา

โดย สำนักงานวิทยทรัพยากร จุฬาลงกรณ์มหาวิทยาลัย

## เครื่องมือที่ใช้

- **HTML + CSS + JavaScript ล้วน** — ไม่มี framework ไม่ต้อง build ไม่ต้องติดตั้งอะไร
- ฟอนต์ Google Fonts: **Trirong** (หัวเรื่อง/ถ้อยคำ ดูสง่า) + **Sarabun** (เนื้อความ)
- ระยะต้นแบบ: เก็บข้อความใน `localStorage` ของเบราว์เซอร์
- รองรับการย้ายขึ้น **Vercel** ได้ทันที (เป็น static site อยู่แล้ว)

## โครงสร้างไฟล์

```
index.html        หน้าเดียวจบ: ส่วนหัว → ฟอร์มเลือกถ้อยคำ+ลงนาม → สมุดลงนาม → ส่วนท้าย
css/style.css     ธีมขาว–ดำ ตัวแปรสีรวมอยู่ที่ :root แก้โทนได้จุดเดียว
js/app.js         ถ้อยคำมาตรฐาน ๘ แบบ, ฟอร์ม, แบ่งหน้า (๕ รายการ/หน้า), ชั้นข้อมูล
.claude/launch.json  คำสั่งรันพรีวิวในเครื่อง (npx serve, พอร์ต 4173)
images/portrait.jpg  พระฉายาลักษณ์ (แสดงเป็นขาวดำอัตโนมัติด้วย CSS)
1.jpg 2.jpg 3.jpg ต้นฉบับถ้อยคำจากสำนักงานราชบัณฑิตยสภา
```

> **ที่มาของพระรูป**: ดาวน์โหลดจาก Wikimedia Commons (ไฟล์ "Princess Bajrakitiyabha.jpg",
> สัญญาอนุญาตแบบ Creative Commons) — หากหน่วยงานมีพระฉายาลักษณ์ทางการที่ได้รับอนุญาต
> ให้นำไฟล์มาแทนที่ `images/portrait.jpg` ได้เลย โดยไม่ต้องแก้โค้ด
> (เว็บจะแปลงเป็นขาวดำให้อัตโนมัติ)

## การทำงาน

1. ผู้ใช้เลือกถ้อยคำ ๑ ใน ๘ แบบ (ตามประกาศสำนักงานราชบัณฑิตยสภา)
2. พิมพ์ชื่อหลังคำว่า "ข้าพระพุทธเจ้า" (บุคคล/คณะบุคคล/หน่วยงาน)
3. กดปุ่ม **"ลงนามถวายความอาลัย"** หรือกด Enter
4. รายการใหม่ขึ้นเป็นรายการแรกของสมุดลงนาม พร้อมแอนิเมชันค่อย ๆ ปรากฏ
5. แสดงหน้าละ ๕ รายการ มีปุ่ม ก่อนหน้า / เลขหน้า / ถัดไป

## รันดูในเครื่อง

เปิด `index.html` ตรง ๆ ได้เลย หรือรันเซิร์ฟเวอร์:

```
npx -y serve -l 4173 .
```

แล้วเปิด http://localhost:4173

## การขึ้นเว็บจริง (ฉบับด่วน ง่าย ฟรี)

เหมาะกับงานอายุ ~3 เดือน ข้อความไม่เกิน ~1,000 รายการ

### ขั้นที่ 1 — ฐานข้อมูล: Google Sheets (ฟรี, 5 นาที)

ทำตามคำแนะนำในหัวไฟล์ [apps-script/Code.gs](apps-script/Code.gs):
สร้าง Google Sheet → วางสคริปต์ → Deploy เป็น Web app → คัดลอก URL (ลงท้าย `/exec`)
มาวางในตัวแปร `API_URL` บนหัวไฟล์ `js/app.js` — จบ ข้อมูลทุกคนเห็นตรงกัน
เปิดดูในชีตได้สด ๆ ลบแถวที่ไม่เหมาะสมได้ และดาวน์โหลดเป็น Excel ได้ทุกเมื่อ

> หมายเหตุ: ตราบใดที่ `API_URL` ยังว่าง เว็บอยู่ใน "โหมดทดลอง" —
> ข้อความเก็บใน localStorage ของเบราว์เซอร์แต่ละเครื่อง ผู้ใช้ต่างเครื่องมองไม่เห็นกัน

### ขั้นที่ 2 — โฮสต์: Vercel (ฟรี, 10 นาที)

1. push โฟลเดอร์นี้ขึ้น GitHub
2. เข้า vercel.com → Add New Project → เลือก repo → Framework เลือก **Other** → Deploy
   (เป็น static site ไม่ต้องตั้งค่าอะไรเพิ่ม)
3. ได้ URL `https://<ชื่อโปรเจกต์>.vercel.app` ใช้งานได้ทันที

ทางเลือกที่ง่ายกว่าอีกถ้าไม่อยากใช้ Vercel: **GitHub Pages**
(Settings → Pages → Deploy from branch) ฟรีเช่นกัน เหมาะกับ static site แบบนี้

### ทางเลือกฐานข้อมูลอื่น (ถ้าต้องการ)

- **SharePoint Excel + Microsoft Graph** — ตามแผนเดิม แต่ต้องลงทะเบียน App ใน
  Azure AD ของจุฬาฯ และเขียน Serverless Function เพิ่ม ใช้เวลาประสานงานมากกว่ามาก
  ไม่แนะนำสำหรับงานด่วนอายุสั้น
- **Supabase / Firebase** — ฟรีเช่นกัน แต่ต้องสมัครบริการเพิ่มและตั้งค่า security rules
  ในขณะที่ Google Sheets ดูข้อมูลง่ายกว่าสำหรับเจ้าหน้าที่
